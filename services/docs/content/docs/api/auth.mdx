Lush splits durable session state from request authorization. The refresh token
is stored only in an HttpOnly cookie, while login and refresh return a
short-lived access JWT. Browser clients cache that access JWT in tab-scoped
`sessionStorage` and send it as `Authorization: Bearer ...` on protected API
requests.

Email verification and password reset are token-only HTTP flows. Their random
bearer tokens are single-use and expiring, and the database stores only token
hashes. Reset completion revokes all sessions for the account. Password-reset
request responses do not reveal whether an account exists.
