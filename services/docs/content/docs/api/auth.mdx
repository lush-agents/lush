Lush splits durable session state from request authorization. The refresh token
is stored only in an HttpOnly cookie, while login and refresh return a
short-lived access JWT. Browser clients cache that access JWT in tab-scoped
`sessionStorage` and send it as `Authorization: Bearer ...` on protected API
requests.
